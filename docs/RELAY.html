RELAY
The context-handoff layer for AI-native teams.

Every AI coding session creates knowledge — the why, the dead-ends, the gotchas, the intent. Today it evaporates the moment the session ends. RELAY captures it, scores it, and makes it portable so the next developer or agent inherits full context instantly.

Build for Builders · Toronto · Jun 26–27 8090 · Cerebras · Backboard · Docker Solo / Deepak Singh Kandari

00The one-liner

RELAY turns a finished AI coding session into a Handoff Capsule — a structured, scored, portable package of intent + decisions + gotchas + next-steps — and injects it into the next person or agent so no context is lost.

8090's Software Factory captures structured context before building (requirements → architecture → work orders). RELAY captures the context created during building — the tribal knowledge their thesis says is lost when experts leave. It is the missing capture layer and feedback loop for the AI-native SDLC.

01The problem — all four hackathon themes, one root cause

Highline Beta asked for tools that improve productivity & flow, code quality & confidence, the junior-developer problem, and the handoff problem. These aren't four problems — they're four symptoms of one thing: context dies at the session boundary.

Handoff

The next dev/agent re-discovers intent & decisions from raw diffs. Tribal knowledge walks out the door.

Junior dev

Juniors lack the senior's mental model — and can't tell a good handoff from a bad one.

Quality & confidence

Undocumented "why" → fear of changing code → regressions and slow reviews.

Flow

Re-reading a codebase to rebuild context is the biggest silent tax on velocity.

02How RELAY works

1 · CAPTURERead the session~/.claude/projects/*.jsonl or the live PTY bridge — the real reasoning, not just the diff.
2 · DISTILLCerebrasInstant structured extraction into the Handoff Capsule schema. Speed = flow.
3 · SCORE6 cognitive dimsRate the handoff's transfer quality — coaches juniors to communicate intent.
4 · STOREBackboardPortable memory that follows the project across sessions, models, people.
5 · INHERITNext agent/devCapsule injected on open → instantly oriented. No re-discovery.

03The money demo — show, don't tell

Side-by-side, live, on stage:

Agent B — WITHOUT capsule

Opens the project cold. Flails. Re-reads files. Asks "what is this for?", re-derives a decision the last session already made, re-hits a gotcha that was already solved.

Agent B — WITH RELAY capsule

Capsule injected from Backboard. Instantly states the intent, the open decision, the gotcha to avoid, and continues exactly where Agent A stopped — first try.

That 30-second contrast is the handoff problem, made visceral. It's the slide 8090 will remember.
