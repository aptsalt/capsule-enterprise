Capture & Feedback Layer · 8090 Software Factory

CAPSULE

Every Claude Code / agent session emits a capsule — a compressed summary of what was important and novel: intent, decisions, gotchas, the mental model, learnings. Capsules surface as a Knowledge-Graph layer on top of 8090's Software Factory and compound — daily — into versioned skills. The result is a reinforcement-learning loop for the enterprise: every session feeds back, and the reward signal is tokens saved downstream.

8090's Software Factory natively runs Requirements → Blueprints → Work Orders → Tests → Knowledge Graph. That pipeline produces software but does not learn from how it was produced. CAPSULE is the capture + feedback layer that closes the loop — turning each session's tacit knowledge into a durable, versioned, reusable enterprise asset.

01

The Reinforcement Loop

A closed enterprise RL loop. Sessions emit capsules (capture); capsules inject context on the next session (feedback); the token delta is the measured reward that decides which capsules get promoted into skill versions.

capture / inject (cyan path) Backboard store skill versioning reward → promotion

Capture (the write half)

  • Session ends or checkpoints → CAPSULE distils only what is important and novel.
  • Token cost of producing the capsule is recorded as the debit.
  • Stored with send_to_llm=false — no generation, no model tokens billed for the write.

Feedback (the read half)

  • Next session opens → relevant capsules + skill memories are retrieved and injected.
  • Tokens not re-derived = the credit.
  • High-reward capsules get minted into a skill version; low-reward ones decay and stop surfacing.

03

Semantic Versioning Rules

Daily, learned capsules are distilled and used to mint a new skill version under major.minor.patch. The bump rule is deterministic — it depends on how the new knowledge changes the contract of the skill.

BumpPatternTriggerExample
patch x.y.Z A gotcha or refinement. Behavior is unchanged; the skill just gets more robust. "add retry on 429 from the migrations API"
minor x.Y.0 A new backward-compatible capability or pattern. Old callers keep working. "skill now also generates rollback scripts"
major X.0.0 Changes how the skill is invoked, or supersedes prior guidance. Breaking. "takes a migration-plan object, not a raw SQL arg"

Immutability & pinning

  • Each minted version is an immutable record — a teammate may have pinned it into their repo.
  • Within a major, versions are additive: adopting @2.3.1 injects the superset of @2.3.0's learnings.
  • A major may supersede: prior guidance is tagged superseded_by:<id> and excluded once the new major is adopted.

Adopt-at-selection

At model- and MCP-selection time the user is shown, per applicable skill:

  • Pinned vs latest version side by side.
  • Token savings projected from adopting the latest.
  • Transfer-score lift — how well its learnings carried to other projects.

The value is shown so the user can self-justify the upgrade.

05

Token Reward — the RL Signal

Token usage is shown everywhere so users derive self-value. The delta between tokens spent to create a capsule and tokens saved downstream by reusing it is the reinforcement reward — measurable, no human labels.

tokens_saved_downstream − tokens_spent_to_create = RL reward
1,840

12,600
credit · saved downstream
+10,760
reward · net token delta

The ledger

  • Debit — tokens spent producing the compression summary at write time.
  • Credit — tokens saved when that capsule / skill memory is injected instead of re-derived.
  • Recorded per capsule and per reuse, so a capsule's reward grows as it's reused.

What the reward drives

  • High positive reward → promoted / minted into a skill version.
  • Low / negative reward → decays, stops being surfaced.
  • Adoption UI shows projected savings so upgrades are self-justifying.

08

Governance

Enterprise-grade controls: tenant isolation is structural, versions are immutable and auditable, writes never leak data into generation, and adoption is an explicit, logged user choice.

isolation

One assistant per tenant (capsule-<ent>). No cross-assistant read path — tenant A can never retrieve tenant B's memory. The wall is structural, not a filter.

immutability

Every minted version is immutable and queryable forever. A major supersedes (tags prior guidance), never mutates in place — so pinned versions stay reproducible.

provenance

Each durable memory carries session_id, capsule_id, and source factory entities — full lineage from learning back to the work order.

no-leak writes

send_to_llm=false on every capture. The write persists and indexes but triggers no generation and bills no model tokens.

explicit adoption

Adopting a new skill version is a user choice at selection time, shown with its projected value and logged — no silent guidance changes mid-flight.

measurable reward

Promotion is driven only by the token delta — no human labels, no opaque scoring. The reward is auditable arithmetic.

The multi-developer enterprise model — promotion, agentic CI, dedup & conflict

The registry is shared across developers (branches dee · ven · saim on master; see REPO-FLOW). A capsule never edits the registry directly — it travels a governed pipeline.

promotion

A capsule clearing the gate (transferScore ≥ threshold OR novelty ≥ 80) is promoted as a proposed version — a PR / commit on a promotion/<skill> staging ref of master, not a direct push. Human / agentic review merges it.

agentic CI

Before publishing, eval.ts runs a multi-sample A/B (N paired runs, new vs current; mean ± stdev + direction-consistency, real token counts) and an LLM-judge regression check of prior capsules. It merges the bump only if the measured reward improves; else it stays proposed or is rejected. CI-for-skills.

dedup

Each capsule carries a semantic signature. Near-identical capsules from different devs merge into one canonical capsule with multiple contributors (provenance keeps every author). Backboard's semantic memory merges the underlying facts.

conflict

"do X" vs "undo X / do Y" on the same skill is a do/undo conflict, resolved by measured reward + recency (ties → human). Contradictory ⇒ major, additive ⇒ minor, refinement ⇒ patch. A merge-ledger records every resolution.

local → EOD → enterprise

A two-tier promotion path. During the day every earned upgrade is committed to the developer's local registry (~/.capsule/local-registry, branch local-deepak, not pushed). The end-of-day job (scripts/eod-promote.ts) pushes and opens one CI-gated PR local-deepak → master against aptsalt/capsule-enterprise-skills (28 skills = 13 real + 15 popular). Merging publishes the day's learning.

purge / retire

The counter-pole to promotion: active → deprecated → archived → purged (scripts/purge-skills.ts, src/lib/purge.ts), flagging ABSORBED / SUPERSEDED / UNUSED / LOW_VALUE / ORPHANED. Dry-run by default, every transition logged to a PURGE-LEDGER; archive moves (recoverable) rather than deletes, and Backboard provenance is never touched.

★

Self-capture proof. CAPSULE captured its own build session (CAP-SESSION-1a6fcc9b) → skill/ui-modularity@1.0.0, promoted into enterprise master via PR #4; dee promoted rest-api-design@1.0.1 + oauth2-jwt-auth@1.0.1. The loop runs on its own code.

10

Technology & Ecosystem Fit

CAPSULE sits at the intersection of three platforms — using each for exactly what it is best at, with no overlap.

8090 · Software Factory

The host surface. CAPSULE is the capture + feedback layer on top of Requirements → Blueprints → Work Orders → Tests → Knowledge Graph. It makes the factory learn.

Backboard · Memory

The durable substrate. Assistant-per-tenant isolation, entity-keyed memory across 17,000+ BYOK models, semantic + LLM-guided recall. #1 LoCoMo / LongMemEval.

Local model · Ollama

The default distiller. Claude Code sessions are the source of learning; qwen2.5-coder:14b compresses them into capsules on-device — private, free, offline. Cerebras is an optional cloud boost for the nightly cross-tenant mint.

★

The throughline: capture once, compound forever. 8090 gives the pipeline, Claude Code generates the work, the local model distils it on your own hardware, Backboard remembers it across models and tenants — and CAPSULE turns the token savings into the reinforcement reward that makes the whole enterprise get smarter every day. Cerebras accelerates the nightly job when you scale past the laptop.
