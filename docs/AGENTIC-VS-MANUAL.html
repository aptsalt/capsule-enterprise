Manual vs Agentic

Same capsule engine. You choose who pulls the trigger.

Both flows read a real ~/.claude/projects/*.jsonl transcript and distill it locally into a Handoff Capsule. The difference is who decides what to keep — you, or the gate.

● Manual

You curate

Hand-pick a session, distill it, and review the capsule yourself before it counts. Full control — ideal for deliberate curation and high-signal handoffs.

How it runs
  1. Pick a sessionCAPSULE lists your real Claude sessions from ~/.claude/projects, newest first.
  2. Distill locallyOne POST /api/capsule reads the transcript and distills it on-device with Ollama. No data leaves the machine.
  3. You review & keepThe capsule renders in full — score, intent, decisions, gotchas, next steps — and you decide to keep it. It's added to Capsules from today.

◆ Agentic · Auto

The gate decides

Every session is auto-distilled, auto-scored, and auto-gated. Only capsules that clear the bar — or carry a unique proposition — are kept. Hands-off, and it scales across a whole team.

How it runs
  1. Ambient auto-distillA non-blocking Stop-hook (settings.json) + scripts/capture-watcher.ts distil every closed session on-device — no one has to press anything.
  2. LLM-judge auto-scoreAn LLM judge scores six transfer dimensions (scoreCapsuleLLM) and a novelty signal (noveltyLLM) — not an authored heuristic. Heuristic fallback if Ollama is down.
  3. Auto-gate → local → EOD PRKept if transferScore ≥ threshold OR novelty ≥ 80 — the bump writes to a local registry (local-deepak); end-of-day, eod-promote.ts opens one CI-gated PR. Agentic CI runs a multi-sample A/B (mean ± stdev) and publishes only if the measured reward improves. Below the bar → dropped, never promoted.

When to use which

Pick the posture that matches the moment.

Manual when judgment matters and volume is low. Agentic when you want coverage without a human in every loop.

If you…● Manual◆ Agentic
Are curating a few high-signal handoffs Best fit — you read and keep each one Overkill — the gate may skip nuance you'd keep
Want every session covered automatically Too manual — you'd have to pick each session Best fit — auto-distill + auto-gate, hands-off

The agentic gate

Kept if it clears the bar — or carries a unique proposition.

Two signals, two bars. Either one is enough to keep a capsule. That's how an ordinary-on-average session with one exceptional insight still gets promoted.

Transfer score72 / 50
Novelty86 / 80

Both signals are LLM-judged on-device: scoreCapsuleLLM rates the six transfer dimensions and noveltyLLM judges how distinctive the finding is — an honest LLM-judged proxy, not an authored heuristic. A capsule can be middling on transfer yet spike on novelty (a rare gotcha, a crisp mental model) and still earn a keep.

# scores are LLM-judged (scoreCapsuleLLM + noveltyLLM)
# the keep rule (default threshold = 50)
keep = transferScore >= threshold
       OR novelty >= 80

# kept → local-deepak → EOD one CI-gated PR
# multi-sample A/B publishes if reward improves
# else → dropped, never promoted

The threshold is yours to set with a slider (0–100, default 50); the novelty bar is fixed at 80. When Ollama is unreachable, a local heuristic fallback distills and scores instead — its scores skew low, so the gate flags it before you trust a skip.

What a kept capsule powers

A capsule isn't an archive. It's recallable context.

Kept capsules feed straight back into the next session.

Toggle Capsule context on, and the composer lights up with a fluorescent-yellow aura — the latest capsule is injected as live context so the next run starts where the last one ended. That's the loop: distill, gate, recall. Whether you curated it by hand or the gate promoted it, a kept capsule shows up here, ready to use.
