Where the models live (in code)

⚡ Local LLM PRIMARY · ON-DEVICE

Distillation runs on your machine first — no key, no network, private by default.

src/lib/cerebras.ts • OLLAMA_MODEL (L14) = qwen2.5-coder:14b • viaOllama() (L65) → POST localhost:11434/api/chat • ollamaHealth()(L98) → GET /api/tags (honest engine) • distill() (L144) → Ollama runs FIRST also: /api/inherit (cold-vs-warm), /api/capsule, scripts/build-real-dataset.ts

☁ Cerebras OPTIONAL · DORMANT

A fully-wired wafer-scale cloud boost — sub-second distillation — that activates only when a key is set.

src/lib/cerebras.ts • CEREBRAS_URL (L8) = api.cerebras.ai/v1/chat/completions • CEREBRAS_MODEL (L9) = llama-3.3-70b • viaCerebras() (L44) → returns null unless CEREBRAS_API_KEY is set (L45) • distill() (L145) → tries Cerebras only if key present

No key today → Ollama does 100% of the work. Set the key and the same pipeline gets a cloud turbo with zero code change.

Docker — three places it fits

① App container

Multi-stage Dockerfile builds the Next.js app and serves it on :3010.

docker build -t capsule .

② Whole stack

docker-compose.yml boots the app + a containerized Ollama — the local model in a box.

docker compose up docker compose exec ollama \ ollama pull qwen2.5-coder:14b

③ Handoff devcontainer

The thesis: a capsule ships its runtime, not just notes. .devcontainer/ = the inheritor gets the exact env + model, pre-wired.

carry the context AND the runtime

Cloud release roadmap

Phase 0YOU ARE HEREshipped

Local-first (hackathon build)

Runs fully on one machine: Ollama 14b distiller, local JSON memory, the whole RL loop. Optional live Backboard + Cerebras via env keys. Docker compose for one-command boot.

Phase 1next

Hosted SaaS · Cerebras-powered

Flip on Cerebras as the cloud distiller (sub-second), Backboard as managed memory, multi-tenant auth + billing. Capture via a Claude Code / VS Code hook + MCP so it's ambient, in the flow.

Phase 2enterprise

Self-host · Docker / Kubernetes

Air-gapped deploy with BYO-model (Ollama cluster or vLLM), SSO, full audit trail, the enterprise skills registry as a governed internal service. CI-for-skills (agentic A/B gate) runs in your cloud.

Phase 3platform

8090 Software Factory native module

CAPSULE installs as a module inside the Factory — the capture + feedback layer that wires into Requirements / Blueprints / Work Orders / Tests via the Knowledge Graph. Org-wide compounding knowledge.

Phase 4network effect

Skill marketplace · federated learning

Share or sell capsule-maxxed skill packs; opt-in cross-org federated learning so a fix discovered once improves everyone — without sharing raw code.
