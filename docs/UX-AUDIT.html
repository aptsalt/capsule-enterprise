CAPSULE vs 8090 — design fidelity review

The CAPSULE chrome and panels (src/components) measured against the 8090 Software Factory reference: a calm, single-blue-accent, document-centric enterprise surface.

Verdict

~70%of the way to the 8090 look

The information architecture, the document canvas, the five side panels and the shared primitives are all in place — and the editor's own SVG tool-icons prove the team can hit 8090's icon quality. But three systemic regressions keep CAPSULE reading as a “hackathon dashboard” rather than calm enterprise SaaS. Fix these and it jumps to ~90%.

  1. Two accents, not one. A full violet (--violet #7c3aed) plus the KG entity palette leaked out of data-viz into chrome — the default Switch tone, active Pill, Adopt CTA, Capsule toggle, engine pill, sidebar dots/badges and avatar gradient. The single biggest “hackathon vs calm enterprise” tell.
  2. Dead document-centric navigation. The center tabs and sidebar file-tree are inert facades — tabs flip local state with no effect, tree rows have no onClick, active doc is a hardcoded constant — so the tabs, tree and editor visibly contradict each other.
  3. Full-color emoji as icons. 💬 ↗ ⤓ ⟳ 🔍 🚀 ⚙ ➤ 🔗 🧠 📄 🎓 render multicolor at inconsistent baselines, shattering 8090's one monochrome line-icon family and adding ~6 uncontrolled colors.

P0 Brand-defining regressions

P0P0-1Color · restraint
One-blue-accent rule broken: violet (and the entity palette) used as interactive chrome
8090Exactly ONE accent (~#2b6cf0) lights active nav, primary buttons, badges and send; everything else neutral.
CAPSULE--violet #7c3aed (plus cyan/green/amber) is promoted into everyday controls — the shared Switch defaults to tone violet, the active Pill is violet, the Adopt CTA is a solid violet button, the composer Capsule toggle is a violet track, SkillCard ships a violet icon chip / version pill / BTN_VIO, and the Skills panel ENTERPRISE switch + “ENTERPRISE · BEST” badge are violet. The eye reads a two-accent product.
FixReserve entity colors strictly for ForceGraph nodes + legend dots. Set Switch default tone='blue' (ui.tsx:126), make violet a graph-only override; active Pill → blue tint; delete BTN_VIO and render Adopt/Update with BTN_PRI; recolor the Capsule toggle and the ENTERPRISE switch/badge to blue or neutral. One accent, everywhere.

P0P0-2Navigation
Document-centric navigation is an inert facade and falls out of sync
8090Center tabs and left file rows are real navigation — clicking swaps the document in the canvas and the active row tints light-blue.
CAPSULETop tabs only flip a local useState with zero downstream effect; sidebar rows render cursor-pointer + hover but have no onClick; ACTIVE_DOC is a hardcoded const that never changes. You can click “Requirements” while the tree and editor headline both stay pinned to “Technical Requirements” — dead, self-contradicting nav.
FixLift the active document into the store (activeDocId) and drive the tab highlight, the sidebar tint and the editor headline/body from it. Wire onClick on each DOC_TREE row and DOC_TAB. Selection must be a single source of truth so tab + tree + canvas always agree.

P0P0-3Iconography
Full-color emoji glyphs break the monochrome line-icon language
8090A single family of small monochrome hairline line-icons, one neutral gray weight + one blue accent.
CAPSULEOS emoji throughout primary chrome — 💬 ↗ ⤓ ⟳ (TopBar), 🔍 (search), 🚀 (capture CTA), ➤ (send), ⚙ (user chip), 🔗 (toolbar), 🧠 📄 🎓 (panels). They render multicolor at inconsistent baselines — a prototype tell that adds ~6 uncontrolled colors.
FixReplace every emoji with the 16–17px stroke SVGs the codebase already authors (DocumentEditor TOOLS). Build a tiny Icon set (search/export/share/history/comment/send/rocket→spark/gear/link) at stroke-1.3 currentColor color:var(--mut) inside the existing IconButton. Replace the colored 💬 and ➤ first.

Roadmap

Quick wins (apply now)

  1. Kill violet in chrome — Switch→blue, delete BTN_VIO, recolor Pill/Adopt/Capsule toggle/dots/avatar. P0-1
  2. Swap emoji → SVG line-icons, starting with 💬 and ➤. P0-3
  3. Restore the neutral status pill; demote the engine pill to gray. P1-1
  4. Add the Ctrl/Cmd-K handler; stop opening Skills per keystroke. P1-5
  5. Friendly capture-error copy + a “Try again” button. P1-6
  6. Unify the sidebar to one accent + one calm score treatment. P1-3
  7. De-decorate version rows to one blue Latest badge. P1-7
  8. Tighten radii to 8–10px and drop hover shadows. P2-7
  9. Centered-icon empty state for the KG explorer. P1-8
  10. Quiet “New Agent” ghost button + add the “+”; flatten avatar. P2-1
  11. Per-action grid icons, round-full blue send, line-icon close. P2-3/4/5

Bigger bets

  1. Make document-centric navigation real — lift activeDocId into the store; drive tabs, tree tint and canvas from it. The biggest trust fix. P0-2
  2. Make the composer a live input — real textarea, working send + Enter, @// mentions, with “Capsule ON” changing what's sent. P1-2
  3. Surface the differentiator in chrome — a calm token-savings ledger pill + a “View provenance” link on the document. P1-9, P2-13
  4. Make panel state & a11y consistent — active-state from every trigger, correct focus restoration, context-aware Escape. P2-9/10/12
  5. Consolidate chrome — route AbTrialsPanel through SidePanel and the Capsule toggle through Switch. P2-6, P2-11
