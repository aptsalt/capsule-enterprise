The problem

Every hackathon theme is the same bug,
wearing a different costume.

Reliability, evaluation, cost, security, multi-agent orchestration — chase any of them far enough and you hit one root cause. The agent that solved it last time isn't here this time.

Reliability
The same gotcha gets re-discovered every session. The fix that shipped on Tuesday is gone by Thursday.
Cost
You pay full token price to re-derive knowledge your team already paid to learn — over and over.
Multi-agent
Agent B can't stand on Agent A's shoulders. There are no shoulders — only a fresh context window.
Evaluation
No way to tell if the system got smarter. Nothing measures whether a lesson actually transferred.
root cause →
Context dies at the session boundary. Enterprises run thousands of agent sessions a day and keep none of the learning. The factory builds; it never remembers.
The insight

8090 captures context before building.
CAPSULE captures what's learned during — and compounds it.

The 8090 Software Factory already turns Requirements → Blueprints → Work Orders → Tests → Knowledge Graph. That's a brilliant forward pass. CAPSULE adds the missing backward pass: the capture + feedback layer that turns each session's output into the next session's intelligence.

Without CAPSULE — a forward pass

Build → ship → forget

Requirements become code. The agent learns idempotency must be amount-scoped, hits a Redis-failover gotcha, finds the fix — then the window closes and the lesson evaporates. Tomorrow, a different agent re-learns it from zero, at full token cost.

learning retained: 0%
With CAPSULE — a closed loop

Build → capture → compound

The same session emits a capsule — the important & novel distilled. It's stored in Backboard, minted into skill/payment-idempotency@2.1.0, and injected into every future session that touches payments. The next agent starts knowing.

learning retained: compounding → 75,000 tokens reclaimed

This is reinforcement learning at the enterprise scale: the action is a build, the reward is the token delta, and the policy is your skill library — getting measurably better every night.

How it works

One loop. Six stages. It never stops turning.

Capture is cheap enough to run on every session twice over: the capsule is distilled on a local model (qwen2.5-coder:14b via Ollama — free, private, no egress), and the write is store-only — send_to_llm=false, no generation, no model tokens billed. Memory follows the entity, not the model.

Every Claude Code / agent session distills only what's important & novel: intent, decisions, gotchas, mental model.
